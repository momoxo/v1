Migration de XOOPS2 à XOOPS Cube 2.1 Legacy

Introduction

Le développement de la base Legacy a pour but de remplacer toute fonctionnalité du module Système. La principale caractéristique de Legacy pour l'utilisateur est la nouvelle structure modulaire qui assure la compatibilité.

Gestion de Modules
  • Vous pouvez gérer les modules depuis le module 'Legacy Système'.
  • L'installation et la gestion de modules ont désormais leur propre interface d'administrative.
  • Les liens se trouvent dans le menu du module Legacy Système.
Gestion d'Utilisateurs
  • 'Utilisateurs', 'Groupes', 'Permissions', 'Avatar' et 'Envoyer email' sont administrés depuis le module utilisateur.
Gestion de Themes
  • Legacy dispose d'un système de rendu (Theme engine) qui assure la compatibilité avec l'environnement smarty de XOOPS2's.
  • Legacy renderer est utilisé pour gérer les themes et templates.

Gestion de Blocs
  • L'environnement XOOPS Cube 2.1 Legacy a vu modifié la fonctionnalité des blocs. Vous devez installer un bloc avant de l'utiliser sur votre site. Le nouveau système Legacy gère ceci de manière simple et visible.
  • Si vous installez un module, les blocs du module sont enregistrés sur XOOPS Cube Legacy, mais ils ne sont pas encore utilisables. Sur la page 'Installer un Bloc' cliquez sur l'icone installer à coté du bloc que vous souhaitez utiliser sur votre site.
  • Vous pouvez éditer, sur les blocs désinstallés, les droits d'accès des groupes, la colonne d'affichage et le module cible. Les blocs désinstallés gardent ces paramêtres. Lorsque vous re-installez le bloc, l'installateur récupère les derniers paramètres.

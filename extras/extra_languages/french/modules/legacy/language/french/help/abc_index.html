XOOPS Cube Legacy : Guide de Démarrage Rapide

Bienvenue sur XOOPS Cube Legacy 2.1, votre site dynamique!
Construisez un portail, un site d'affaires, votre site personnel, de votre famille ou de votre passe-temps.

XOOPS Cube est la structure de base pour vos modules, gérés depuis le module 'Legacy'.
XOOPS Cube Legacy est distribué avec des ressources minimales.
Vous pouvez télécharger, selon vos besoins, divers modules & themes.
Lisez les pages de ce Guide Rapide pour en savoir plus sur Cube Legacy 2.1.


Démarrer

Avant tout, il vous faut assurer la sécurité de votre site.
Ensuite, vous pourrez commencer à construire votre nouveau site web.
Allez dans le panneau de contrôle de votre site et vérifiez qu'il n'existe pas de messages d'alerte dans la page d'administration.

Alerte sur la présence du repértoire d'installation

Ceci est plutôt risqué! Retirez le dossier 'install' de votre espace public!
Un utilisateur malveillant peut accéder à l'installateur de votre site.
Il peut réinstaller XOOPS Cube sur votre site pour devenir l'administrateur.
Vous pouvez supprimer ce dossier 'install' si votre XOOPS Cube Legacy est installé avec succès.
Mais si vous ne savez pas comment faire, veuillez le signaler à l'administrateur du serveur.

Alerte sur l'écriture possible du fichier 'mainfile.php'

Le message vous avertit qu'une attaque malveillante peut récrire votre mainfile.php.
Vous devez interdire l'écriture de 'mainfile.php' pour vous préserver de ce genre d'attaque.
Cette opération peut changer selon le type de système opératif de votre serveur.
Sur les serveurs linux, unix, ou de la même famille, changez les permissions en 644.
Si votre serveur est de la famille Windows, changez le fichier en mode 'lecture seule'.
Si vous avez toujours un message d'alerte, veuillez le signaler à l'administrateur du serveur.

OK? Si le panneau de contrôle n'affiche plus de messages d'erreurs, passons à l'étape suivante!

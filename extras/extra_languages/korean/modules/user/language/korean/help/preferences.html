사용자 모듈 - 일반설정

신규 회원의 등록을 허용함?

예 를 선택하시면 신규 회원 등록을 허용하게 됩니다. 아니요 를 선택하신 경우엔 관리섹션에서 관리자가 직접 회원추가 작업을 하셔야 합니다.

패스워드의 최저 문자수

패스워드의 최저 문자수를 지정합니다. 짧은 패스워드보다는 긴 패스워드가 더 안전할 수 있습니다만 패스워드의 길이만이 좋은 패스워드의 조건은 아님에 유의하시기 바랍니다. 기본적으로 5문자이상의 길이로 지정하실 것을 권합니다.

사용자아이디의 최저 문자수

사용자아이디의 최저 문자수를 지정합니다. 이 옵션은 보안과는 별로 관계가 없으니 님의 사이트에 맞는 적당한 값으로 결정하시면 됩니다.

등록회원의 자기 메일주소 변경을 허용함?

사용자가 회원정보 편집 페이지(edituser.php)에서 자신의 메일주소를 임의로 수정하는 것을 허가할 지 정합니다. 참고로 사용자가 임의로 메일주소를 변경했을 때 확인메일을 따로 보내진 않습니다.

신규 회원의 등록이 있는 경우 메일로 통지/통보 받음?

이 옵션을 예 로 설정하시면 아래의 옵션에서 지정한 그룹에게 신규 사용자 등록과 관련된 통지메일을 보내게 됩니다.

신규 회원 등록 통지/통보 메일을 받을 그룹을 설정해 주세요!

위 옵션에서 예를 선택했을 경우 이 옵션에서 지정한 그룹에게 신규 사용자 등록과 관련된 통지메일을 보냅니다. 일반적으로 사용자그룹을 지정하시면 됩니다.

신규 회원 계정의 활성화 방식을 선택해 주세요!

등록후 사용자는 계정 활성화작업을 해야만 합니다. 계정을 활성화해야만 로그인이 가능하게 됩니다. 이에는 다음의 몇가지 방법이 있습니다.

계정 활성화를 위해 회원 자신의 확인이 필요(추천)

일반적으로 가장 많이 사용되는 방식으로 사용자에게 계정 활성화용 URL이 적힌 메일을 보내어 이를 통해 사용자가 직접 계정활성화를 할 수 있게 합니다. 전세계적으로 가장 많이 사용되는 방식이니 잘 모르실 경우엔 이 방식을 선택하시면 무난하다고 할 수 있습니다.

자동적으로 계정을 활성화함

이 옵션은 계정활성화 기능을 무효화합니다. 즉 사용자는 등록과 동시에 계정이 자동으로 활성화되어 바로 로그인 가능상태가 됩니다. 이 방식의 경우 BOT에 주의하실 필요가 있습니다 BOT으로 자동으로 계정을 만들어 스팸을 뿌리는 경우가 한 예입니다. 따라서 이 방식은 인트라넷과 같은 닫힌 네트워크 환경에서 이용하시는 편이 좋습니다.

관리자가 계정 활성화 여부를 결정

이 옵션은 특정 그룹에게 계정 활성화 작업을 위임합니다. 즉 위임받은 특정 그룹이 확인후 수동으로 계정활성화 작업을 하게 되는 것입니다.( 일반적으로 관리자그룹! 물론 옵션에서 다른 그룹을 지정할 수도 있습니다.)

아바타 그림파일의 업로드를 허용함?

사용자에게 아바타 그림파일의 업로드를 허용할지를 정합니다. 아니요를 선택한 경우엔 관리자가 준비한 시스템아바타를 이용하게 됩니다.

아바타 업로드권한을 위한 최저 요구 투고수

등록회원이 자기의 아바타를 업로드할 권한을 얻기위해 필요한 최저 투고수를 입력해 주세요.(커스텀 아바타업로드를 허용한 경우만)

투고수 제한을 두고 싶지 않을 경우 0 으로 두시면 됩니다.

아바타 그림파일의 최대허용 폭 (pixel)

아바타 그림파일의 최대허용 폭 (pixel)을 지정합니다.(커스텀아바타 업로드를 허용한 경우만)

지정값을 초과한 경우 에러가 발생하게 됩니다.

아바타 그림파일의 최대허용 높이 (pixel)

아바타 그림파일의 최대허용 높이 (pixel)를 지정합니다.(커스텀아바타 업로드를 허용한 경우만)

지정값을 초과한 경우 에러가 발생하게 됩니다.

아바타 그림파일의 최대허용 파일사이즈 (byte)

아바타 그림파일의 최대허용 파일사이즈 (byte)를 지정합니다.(커스텀아바타 업로드를 허용한 경우만)

지정값을 초과한 경우 에러가 발생하게 됩니다.

등록회원의 자기 계정삭제를 허용(탈퇴허용)?

등록회원의 자기 계정삭제를 허용할지 정합니다. 예로 지정한 경우 사용자는 프로필페이지(userinfo.php)의 관련 링크를 통해 자신의 계정을 삭제할 수 있게 됩니다.(관련 템플렛: user_delete.html, user_delete_success.html )

계정삭제전 확인메세지

계정 삭제시 확인용으로 표시되는 메세지입니다.(자기 계정삭제를 허용한 경우에만)

사용자명으로 사용할 수 없는 문자열

이를 지정하면 사용자는 해당되는 문자열을 사용자명에 사용할 수 없게 됩니다.(각각의 문자열은 | 로 구분, 대소문자는 구별하지 않음, 정규식 사용가능)

사용자의 회원등록시에만 적용됩니다. 관리자의 회원관리작업시에는 적용되지 않습니다.

회원의 메일주소로서 사용할 수 없는 문자열

이를 지정하면 사용자는 해당되는 문자열을 메일주소에 사용할 수 없게 됩니다.(각각의 문자열은 | 로 구분, 대소문자는 구별하지 않음, 정규식 사용가능)

이용약관(disclaimer)을 표시?

아래에서 지정한 사이트 이용약관을 신규등록 페이지에 표시할지 정합니다.

이용약관(Registration disclaimer)

신규등록 페이지에 표시될 사이트 이용약관을 작성하시면 됩니다.(이용약관 표시여부를 예로 지정한 경우만)

등록회원에게 할당할 쿠키의 이름

할당할 브라우저용 쿠키의 이름을 지정합니다. 대부분의 경우 기본값을 사용하시면 됩니다.

로그인에 SSL을 사용?

SSL 로그인을 사용하시려면 추가 전용 스크립트를 준비하실 필요가 있습니다. 예 를 선택하시면 로그인 블록에 아래 옵션항목에서 지정한 SSL 로그인용 링크가 표시되게 됩니다. 전용 스크립트는 "/extra" 디렉토리에 들어 있습니다.(SSL관련 전문적 사항은 관련 서적을 참고하시기 바랍니다.)

SSL 로그인시에 사용될 Post 변수명

SSL 로그인시에 사용될 Post 변수명을 지정합니다.

SSL 로그인 페이지의 URL

SSL 로그인 페이지의 URL을 지정합니다. 준비하신 전용 스크립트의 URL을 지정하시면 됩니다.

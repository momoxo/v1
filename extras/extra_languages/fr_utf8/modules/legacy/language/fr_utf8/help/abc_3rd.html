La Troisième Etape - Installer des modules

La Troisième Etape (1) - Comment installer des modules

Les modules sont le facteur supplémentaire le plus important dans XOOPS Cube Legacy.
XOOPS Cube Legacy est juste la plate-forme commune pour les utilisateurs de XOOPS.
Comme une console de jeu sans jeux n'est utile à personne, XOOPS Cube Legacy sans les modules ne serait pas non plus.
Votre site sera complet, une fois installé quelques modules.

Vous avez besoin de modules supplémentaires et d'un client FTP pour les placersur votre serveur.
Commençons par obtennir les modules. XOOPS Cube Legacy 2.1 peut utiliser les modules suivants.

  • Les Modules de XOOPS Cube Legacy 2.1
  • Les Modules de XOOPS 2.0.x JP
  • Les Modules de XOOPS 2.0.9

XOOPS Cube Legacy a des caractéristiques compatibles avec XOOPS2 JP.
La plupart des modules XOOPS2 JP et XOOPS 2.0.9 sont donc traités par XOOPS Cube Legacy.
Mais, cette compatibilité n'est pas encore parfaite.

Télécharger de nouveaux modules

Où trouver les informations et télécharger les modules ?
xoopscube.org et Wiki page sont un bon départ, de même que pour les thèmes.
Soyez prudent lors du téléchargement de fichiers de modules et autres thèmes.
Vous pouvez avoir également besoin d'outils de décompression, parce que la plupart des modules sont distribués sous forme de fichiers compressés.

Installer des modules sur votre site

Placez le dossier du module décompressé dans le dossier 'modules' de votre XOOPS Cube Legacy.
Le dossier doit inclure un fichier 'xoops_la version. php' et, ou 'packet.ini.php'.
Gardez à l'esprit l'arborescence du dossier suivant.

Attention ! Une erreur très fréquente des utilisateurs est de télécharger un dossier supplémentaire, comme l'exemple suivant.

La Troisième Etape (2) - Installer le module forum

Votre projet de site web consiste, par exemple, à publier des articles et des forums de discussion.
Il existe plusieurs modules de forum comme de livres d'or ou comme tout autre type de module.
Vous pouvez installer votre module préféré. Mais, cette fois, nous installons ce bon vieux module 'newbb'.
C'est un module facile à installer et à expliquer. Newbb était le module de forum standard de XOOPS2.
A présent, vous pouvez obtenir un des successeurs : newbb2, xhnewbb et d3forum.

Télécharger newbb

Télécharger l'archive et décompresser. Ensuite, placer 'newbb' dans le dossier /modules de votre site XOOPS Cube Legacy.

Installer le module

Cliquez sur le lien "Installer un Module" du module Legacy dans le bloc gauche du panneau de contrôle.

Vous voyez la liste de modules désinstallés.

Cliquez sur l'icône installer de la ligne indiquant le module newbb.

Le message d'installation s'affiche. Et vous n'avez plus qu'à confirmer l'installation de newbb en cliquant sur "Installer".

Options du module 'newbb'

Allez sur la page d'accueil et vérifiez que newbb est bien installé.
Mais, vous ne pouvez pas encore utiliser newbb, parce que newbb n'est pas encore configuré.
La plupart des modules ont besoin de paramètres additionnels pour fonctionner.
Commencez par lire la documentation des module.

Pour expliquer le principe d'installation d'un module, nous avons pris comme exemple le module Newbb qui gère un forum standard.
Cliquez sur newbb (Forum) dans le menu latéral du panneau de contrôle.
Ensuite cliquez sur le lien "Catégorie". Les catégories peuvent contenir plusieurs forums.

Créer la catégorie "Ligue 1", la catégorie "Ligue 2" et la catégorie "Club Local".

Á présent, il vous est possible d'ajouter des forums dans ces catégories.
Cliquez sur le lien "Ajouter un Forum" du bloc newbb à gauche du panneau de contrôle.

Dans la catégorie "Ligue 1", créez les forums "Championnat 2006", "Equipes" et le forum "Matchs".
Ensuite créez les mêmes forums dans la catégorie "Ligue 2" et "Club Local".

Newbb est prêt à gèrer les discussion sur ces sujets!
Allez à la page d'accueil pour confirmer.

La Troisième Etape (3) - Installer le module Articles

Un autre partie du projet consiste à installer un module pour publiers des articles.
L'administrateur peut ainsi poster l'information importante - les nouvelles du site, les articles de joueurs, du club ou d'ailleurs - depuis le module 'Articles'.

De même que pour le module de forum, il y a beaucoup de modules d'articles.
Et comme pour newbb, nous installons le module d'articles par défaut 'News'.
Le module "News" était le module d'articles standard de XOOPS2.
Le code est ancien et dépassé. Mais, ce module est très facile à installer et à expliquer aussi.
Vous trouverez le module d'articles qui vous convient pour remplacer ce module d'Articles.

Paramètres des Articles

Vous avez besoin de configurer le module Articles comme le forum.

Cliquez sur le lien "Sujet" de module Articles dans le bloc gauche du panneau de contrôle.

Ensuite, ajouter le sujet "Les Nouvelles du Site" et "Articles de Joueurs".

Vous pouvez effacer le sujet "XOOPS", parce que ce module d'Articles crée le sujet "XOOPS" par défaut.
Pour cela, allez dans les "Sujets", sélectionnez "XOOPS" pour modifier.

Finalement, cliquez le bouton "Effacer".

La Troisième Etape (4) - Comment contrôler l'ordre (ou poids) des modules

Sur la page d'accueil vous pouvez vérifier que les deux modules sont bien installés.
Vous pouvez remarquer également que module Forum est au dessus du module Articles, dans le bloc du menu principal.

De même que pour les blocs, il est possible de contrôler l'ordre d'affichage depuis la "Gestion des Modules".
Cliquez sur "Gestion des Modules" du module Legacy dans le bloc gauche du panneau de contrôle.

Vous voyez la liste de modules installés.

Ensuite, sur la colonne 'ordre', ajoutez la valeur 5 au module newbb. Et, la valeur 10 au module Articles.

Cliquez ensuite sur le bouton "Mise à jour".
Confirmez les changements plus haut et cliquez encore sur le bouton "Mise à jour".

NOTES : Comment renommer des modules

L'espace d'affichage du bloc du menu principal n'est pas suffisant.
Ou les noms des modules ne sont pas clairs pour les visiteurs.
Vous pouvez les renommer dans la gestion de modules.

Nom du Dossier

Les modules ont deux genres de noms, le "nom" et "dirname".
Le titre "nom" peut être changé librement, comme vous avez déjà vu pour renommer les modules.
Alors que "dirname » est le terme technique de XOOPS, qui signifie "nom du dossier".
Et pour la plupart des modules il est impossible de renommer leur "dirname".
Garder à l'esprit comme un principe fondamental de XOOPS Cube Legacy.

La Troisième Etape (5) - Terminer la première page
Ajouter de nouveaux blocs du forum et articles

Ajoutons enfin de nouveaux blocs à la première page pour afficher les derniers messages des utilisateurs.
Allez sur "Installer un Bloc".
Vous voyez les nouveaux blocs du module newbb et du module Articles.
La plupart des modules ont leurs propres blocs.
En installant les modules il devient possible d'installer ces blocs.
Maintenant, installez le bloc du module newbb "Sujets Récents".

Installez aussi le bloc du module Articles "Articles Récents".

Spécifiez comme suit.

  • La position est "Bloc Centre-Centre".
  • Les modules cibles sont uniquement "Toppage".
  • Les groupes cibles sont tous les groupes.

Contrôler l'ordre d'affichage des blocs dans la colonne de centre

Cliquez sur "Gestion des Blocs".
Ajoutez alors la valeur 5 dans la colonne 'ordre' du bloc personnalisé "Bienvenue".
La valeur 10 sur le bloc "Articles Récents" et la valeur 15 sur "Sujets Récents".
Finalement cliquez sur le bouton "Valider".

Conclusions

L'archive standard de XOOPS Cube Legacy ne satisfait pas les besoins des utilisateurs et administrateur.
Mais, vous pouvez obtenir d'autres modules puissants et les installer.
Et rappelez-vous que les blocs peuvent être installés après l'installation des modules.

Migration de XOOPS2 à XOOPS Cube 2.1 Legacy

Introduction

Toute fonctionnalité du module Système est active pendant la phase de développement de XOOPS Cube 2.1 Vous aurez besoin de ces anciennes fonctionnalités pour assure une certaine compatibilité.

Ces caractéristiques seront portés sur le module Legacy jusqu'à la version stable.

Gestion de Modules
  • Vous pouvez gérer les modules depuis le module 'Legacy Système'.
  • 'Installer un module' et la gestion de 'Modules' ont désormais leurs propres pages d'administration.
  • Cliquez sur le lien du menu menu 'Modules' pour gérer les modules installés. Cliquez sur le menu 'Installer un Module' pour installer de nouveaux modules.
  • Le système d'installation du module Legacy a toujours quelques bogues. Et, ne met pas en application toutes les fonctions du système d'installation du module Système.
Gestion d'Utilisateurs
  • 'Utilisateurs', 'Groupes', 'Permissions', 'Avatar' et 'Envoyer email' se retrouvent dans le module utilisateur.
Gestion de Themes
  • Vous trouvez ici le système de rendu (Theme engine) utilisé pour gérer vos themes.
  • Nous pensons que la gestion des templates est une propriété du système de rendu. Cependant, la gestion des templates est placé dans le module 'Legacy renderer' qui assure la compatibilité avec l'environnement smarty de XOOPS2's.
nouveau!
Gestion de Blocs
  • L'environnement XOOPS Cube 2.1 Legacy a vu modifié la fonctionnalité des blocs. Vous devez installer un bloc avant de l'utiliser sur votre site. Le nouveau système Legacy gère ceci de manière simple et visible.
  • Si vousinstallez un module, les blocks du module sont enregistrés sur XOOPS Cube + Legacy, mais ils ne sont pas encore utilisables. Sur la page 'Installer un Bloc' cliquez sur l'icone installer àcoté du bloc que vous souhaitez utiliser sur votre site.
  • If you uninstall the block, it returns to the list of 'Install block' page.
  • Vous pouvez éditer, sur les blocs désinstallés, les droits d'accès des groupes, la colonne d'affichage et le module cible. Les blocs désinstallés gardent ces paramêtres. Lorsque vous re-installez le bloc, l'installateur récupère les derniers paramètres.

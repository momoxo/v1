Module Utilisateur - Préférences

Autoriser l'enregistrement de nouveaux utilisateurs?

Spécifie si les utilisateurs anonymes peuvent s'enregistrer sur votre site. En cas d'OUI, registre.php est disponible, et le Bloc login affiche le lien registrer.php. Même si l' option coché est Non, vous pouvez ajouter un nouvel utilisateur depuis laGestion d'utilisateurs.

Longueur minimum requise pour le mot de passe

Spécifie la longueur minimum de mot de passe. Un mot de passe long est plus difficle à deviner. Bien sûr, la longueur n'est pas le seul facteur d'un mot de passe fort. Mais, vous devriez spécifier cinq ou plus.

Longueur minimum requise pour le nom d'utilisateur

Spécifie la longueur minimum d'identifiant. Cette option n'est pas un critère de sécurité. Décidez selon vos préférences.

Autoriser les utilisateurs à changer leur adresse e-mail?

Spécifie si les utilisateurs peuvent changer leur adresse e-mail dans l'éditeur du profil utilisateur (edituser.php). En fait, ce module n'envoie pa de confirmation aux utilisateurs qui change leur adresse.

Notifier par email lors de l'enregistrement d'un nouveau utilisateur?

Dans le cas où cette option est Oui, le groupe spécifié par l'option suivante peut recevoir le courrier de notification des nouveaux enregistrements d'utilisateurs.

Choisissez le groupe auquel l'email d'activation doit être envoyé

Spécifie le groupe qui recevra le courrier de notification dans le cas où l'option précédente est Oui. Le groupe de webmestre est le plus approbon d'habitude pour cette option.

Sélectionnez le type d'activation des nouveaux utilisateurs enregistrés

Après l'enregistrement, un utilisateur doit être activé. Il y a plusieurs façons pour activer des utilisateurs.

Activation par l'utilisateur requise

C'est le cadre le plus populaire dans l'univers des systèmes de site web. Un utilisateur obtient un e-mail d'activation quand il s'enregistre. Ensuite il suffit de cliquer sur le lien d'activation de l'e-mail. C'est l emeilleur choix dans la plupart des cas.

Activation automatique

Dans des termes simples, il n'y a pas vraiment de mécanisme d'activation, tout utilisateur est activé tout de suite. Mais, si vous utilisez ceci, vous devez faire l'attention aux BOT. Les BOT peuvent s'enregistrer et vous soumettre automatiquement du SPAM. Donc, ce choix convient à un réseau fermé.

Activation par les administrateurs

Un utilisateur peut être seulement activé par l'approbation directe du groupe spécifié dans l'option suivante. Cette sélection devient une charge de travail assidu pour les activateurs.

Sélectionner le niveau de restriction pour le filtrage des noms d'utilisateur

Limite le type de caractères dans le nom d'utilisateur.

Strict (uniquement alpha-numérique)

Avec ce choix, votre site permet les caractères suivants pour le nom d'utilisateur.

  • Alphabet sans être sensible à la case.
  • Nombres.
  • Souligné (_).
  • Trait d'union (-).
Moyen

Avec ce choix, votre site permet les caractères suivants pour le nom d'utilisateur.

  • Alphabet sans être sensible à la case.
  • Nombres.
  • Souligné (_).
  • Trait d'union (-).
  • Le caractère moins que (<).
  • Le caractère plus grand que (>).
  • Virgule (,).
  • Point (.).
  • Dollar ($).
  • Pourcent (%).
  • Dièse (#).
  • Arrobas (@).
  • Point d'exclamation (!).
  • Citation seule (›).
  • Citation double (").
Faible (récommendé pour les caractères multi-bits)

Avec ce choix, votre site permet tous les caractères sauf le code contrôle. Mais, cette option peut causer des ennuis avec quelques modules. Donc, ce choixe n'est pas récommendé pour ce module.

Autoriser l'utilisateur a envoyer un avatar personnalisé?

Spécifie si un utilisateur peut télécharger son dossier d'image préféré comme son avatar à ce site. Même si le cas de Non, un utilisateur peut choisir son avatar des avatars de système qu'administrateurs préparent.

Envois minimum requis

Si permettre l'avatar à personnalisé télécharge, vous n'avez pas besoin de toucher cette option.

Ceci est le nombre minimum de postes qu'utilisateurs peuvent obtenir la droite de télécharger l'avatar personnalisé. Il peut avoir a le goût pauvre. Zéro est bon.

Largeur maxi de l'image avatar (pixels))

Si vous ne permettez pas l'avatar à personnalisé télécharge, vous n'avez pas besoin de toucher cette option.

Spécifie la largeur de limite pour l'avatar personnalisé. Dans le cas où un utilisateur télécharge une image plus large que cette valeur, ce module affiche des messages d'erreur.

Hauteur maxi de l'image avatar (pixels)

Si vous ne permettez pas l'avatar à personnalisé télécharge, vous n'avez pas besoin de toucher cette option.

Spécifie la hauteur de limite pour l'avatar personnalisé. Dans le cas où un utilisateur télécharge une plus haute image que cette valeur, ce module affiche des messages d'erreur.

Taille maxi de l'image avatar (Octets)

Si vous ne permettez pas l'avatar à personnalisé télécharge, vous n'avez pas besoin de toucher cette option.

Spécifie le filesize de limite pour l'avatar personnalisé. Dans le cas où un utilisateur télécharge un plus grand dossier d'image que cette valeur, ce module affiche des messages d'erreur.

Autoriser les utilisateurs à supprimer leur compte?

Spécifie si les utilisateurs peuvent effacer le propre compte dans le profil d'utilisateur édite. En cas d'Oui, les utilisateurs peuvent aller au compte efface le dialogue par le lien du profil d'utilisateur (userinfo.php). Parce que ces dialogues d'effacer sont très simples, vous devez éditer des gabarits d'eux -- -l'utilisateur_efface. le html, l'utilisateur_efface_le succès. le html.

Message de confirmation de la suppression du compte

Ceci est le message affiché dans le compte efface le dialogue. Si vous ne permettez pas d'utilisateur pour effacer le propre compte, vous n'avez pas besoin de toucher cette option.

Entrez les noms qui ne doivent pas être utilisés par les les utilisateurs

Ceci est des modèles que votre site ne permet pas comme l'identifiant. Séparer chaque avec un |. Ceci est le cas insensible et regex a rendu capable. En plus, cette règle ne s'applique pas à la direction d'utilisateur dans le tableau de commande avant.

Si vous ne permettez pas d'utilisateurs anonymes pour enregistrer à ce site, vous n'avez pas besoin de toucher cette option.

Entrez les e-mails qui ne doivent pas être employés dans les profils utilisateurs

Afficher les conditions d'utilisation du site?

Choisissez OUI pour afficher le disclaimer dans la page d'enregistrement

Conditions d'Utilisation

Ceci est le refus montré dans l'enregistrement d'utilisateur pour l'utilisateur anonyme. Si l'option précédente est Non, vous n'avez pas besoin de toucher cette option.

Nom pour le cookie utilisateur.

Spécifie le nom de petit gâteau pour les navigateurs web. Dans la plupart de cas, vous n'avez pas besoin de changer ceci.

Utiliser une connexion SSL?

Vous devez installer le manuscrit de login supplémentaire spécial pour activer le login de SSL. En cas d'OUI, Bloc de login Montre dont le nom au lien est « le Login de SSL » spécifié par les options prochaines. Ces options sont pour la compatibilité avec XOOPS2 JP. Peut-être, vous pouvez trouver de meilleures solutions que ceci pour le Login de SSL. Le manuscrit est dans « /extra » annuaire.

Nom de la variable SSL

Cette option est pour le Login de SSL, aussi. Spécifie un nom de la variable de POSTE-DEMANDE.

URL pour la page de la connexion SSL

Cette option est pour le Login de SSL, aussi. Spécifie l'URL où vous avez installé le manuscrit de login supplémentaire spécial.

Module Utilisateur - Principes de base

Avant tout, vous devez comprendre les cinq choses élémentaires qui sont Utilisateur, Groupe, Rang, Avatar, Permission. Cette partie vous explique les principes de bases. Pour plus d'informations, vous devriez lire chacun des sections.

Utilisateur

Ceci veut dire un compte d'utilisateur. Un compte utilisateur a les informations sur l'authentification de Xoops et les champs de profil. L'adresse email, Uname, et le mot de passe sont nécessaires pour la connection au site. Les comptes d'utilisateur sont ajoutés par enregistrement dans l'administration et par l'utilisateur.

Groupe

Signifie un groupe de comptes d'utilisateur. Un compte utilisateur peut appartenir à plusieurs groupes. Un groupe a des permissions de modules. Dans ce module, les permissions ne sont pas données à des utilisateurs mais à des groupes. Des permissions des utilisateurs sont définies par les groupes aux quels les utilisateurs appartiennent.

Vous pouvez créer des groupes des deux manières, la gestion de permissions et la liste d'utilisateurs. Mais, vous devez savoir qu'il y a trois groupes fixes. Ces groupes spéciaux ont été créés lors de l'installation, et il est impossible de les supprimer.

Webmasters

Les utilisateurs Webmasters ont des permissions pour tout faire. En d'autres termes, les utilisateurs qui peuvent appartenir à ce groupe sont les webmasters uniquement. Vous devez faire attention au fait que ce groupe peut tout faire. Dans le cas où vous pensez laisser à vos amis une partie de la gestion du site, ne placez pas vos amis dans ce groupe.

Il est important de se rappeler que ce groupe a la permission de tout faire.

Les utilisateurs enrregistrés

Les utilisateurs enregistrés font partie de ce groupe. L'enregistrement d'un nouveau utilisateur utilise cce groupe par défaut. Dans aucun cas, vous devez donner des permissions d'administration à ce groupe.

Les Utilisateurs anonymes

Les accès sans une connection utilisateur sont gérés par les permissions de ce groupe. Ce groupe est un peu spécial, un genre de groupe virtuel. Ce groupe ne devrait pas avoir de membres.

Rang des Utilisateurs

Le Rang est une partie du profil utilisateur. Le Rang comprend l'image symbole et le titre. Un rang d'un utilisateur est décidé par le nombre de ses contributions. Ce dispositif est un élément plutôt ludique et amusant du site. Par conséquent, vous n'avez pas besoin de vous inquiéter sur les conséquences du Rang sur l'authentification de Xoops.

Avatar

L'avatar est un autre élément du profil utilisateur. C'est une identité graphique personnelle. Il y a deux genres d'avatars. L'un d'entre eux est l'Avatar du système, que les administrateurs ont préparent et fournit. L'autre est l'Avatar personnalisé, une image que les utilisateurs téléchargent sur le site.

Les Permissions

Les permissions d'action des groupes. Ce module permet d'éditer les permissions debase. Si vous voulez configurer les permissions spécifiques d'un module, vous devez utiliser l'éditeur de permissions propre a chaque module.

Les permissions d'un utilisateur sont décidées par l'union de toutes les permissions des groupe aux quels l'utilisateur appartient.
